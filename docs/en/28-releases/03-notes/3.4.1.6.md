## Features

1. Added an XNODE management page to taos-explorer
2. Added support for mandatory access control (MAC)
3. Added support for a mandatory separation-of-duties permission model
4. Added a connector security development guide and security examples
5. Set via to none by default when creating a task for the first time

## Enhancements

1. Optimized communication logging between xnoded and taosX
2. When creating taosX tasks in taos-explorer, users are automatically redirected to the XNODE creation page if no XNODE is available
3. In Explorer DataIn, the default empty tag column is automatically removed when a supertable data column is moved to the tag column
4. Improved SQLAlchemy compatibility in taos-ws-py
5. Added support in the Rust connector for subscribing to batch tag update messages
6. Added daylight saving time conversion support to the TO_ISO8601 function
7. Added DECIMAL and BLOB support to parameter binding in the Rust connector
8. Added minute-level timezone offset support for the TZH format in the to_char and to_timestamp functions
9. Added support for importing and exporting taosX configuration files through taos-explorer
10. Added automatic reconnection support for the AVEVA Historian data source in taosX after disconnection
11. Added the -H option to taos shell to display BINARY strings containing non-printable characters in hexadecimal format (0x...), disabled by default
12. Added the syncAssignedCheckAppliedGap parameter. In dual-replica mode, when a node is assigned as leader, this parameter controls the threshold gap between the follower's applied index and commit index to prevent the follower from entering the restoring state before catching up
13. Optimized the output format of show table distribute and added the INFORMATION_SCHEMA.INS_DISK_USAGE system table for querying table data distribution
14. In STMT2 write mode, child-table metadata changes no longer return the TSDB_CODE_NEED_RETRY error
15. Added support for modifying encrypted configuration files through the taosk command
16. Tables used by subscriptions can now be dropped. After dropping and recreating the table, running reload topic restores the subscription
17. Added DECIMAL column support for virtual tables
18. Interval windows now support partitioning based on the timezone carried by the client
19. Optimized the truncation logic when the last WAL file is corrupted
20. Improved cross-origin security in taos-explorer

## Fixes

1. Fixed slow loading of the Explorer DataIn tasks page when many tasks exist
2. Fixed conflicts between expired data deletion (trim) and shared-storage migration operations
3. Fixed failures when writing monitoring information to the log database
4. Fixed inaccurate SQL execution times returned by taos-explorer
5. Fixed crashes caused by executing certain functions in queries without a FROM clause
6. Fixed backup point generation errors in taosX when no data is available
7. Fixed errors when writing stream computing results to an existing regular table as the result table
8. Fixed an OOM issue caused by incorrect memory release logic when using %%tbname in stream queries under large-write workloads
9. Fixed high memory usage in count_window when reading historical data in stream computing
10. Fixed crashes and incorrect results in the group_concat function
11. Fixed the issue where filter conditions did not take effect when configuring taosX tasks in taos-explorer
12. Fixed the issue where taosdump returned empty data without throwing an exception when the engine service became unavailable during WebSocket reads
13. Fixed the issue where the Pulsar data source in taosX remained in the creating state because data splitting is unsupported
14. Fixed errors when creating streams in nanosecond-precision databases if fixed values were used for TAG columns in stream output tables
15. Fixed the issue where the Windows installer could not shut down processes gracefully
16. Fixed the issue where the authentication parameter input box was not displayed when Custom Authentication was selected for the Pulsar data source in taosX
17. Fixed the intermittent failure to stop xnoded child processes on Windows
18. Fixed significant performance degradation when taosdump exported large-volume subtables
19. Fixed the issue where the queryTableNotExistAsEmpty parameter did not take effect when accessed through taosAdapter
20. Fixed incorrect auto-increment behavior when taosX generated timestamp column data with composite primary keys for Kafka data sources
21. Fixed a latent risk caused by inconsistent column counts between virtual subtables and virtual supertables in the schema
22. Fixed the issue where taosX did not display an error and could not recover or retry when fetching sample data for SparkPlugB
23. Fixed inconsistencies between the execution plan and actual execution when CASE WHEN referenced the primary key as a filter condition
24. Fixed incorrect path handling when generating keys with the taosk tool
25. Fixed crashes caused by type-conversion expressions such as not (cast(null as varbinary) between now() and ts) in queries
26. Fixed OOM crashes caused by an uncontrolled upper limit on the number of sliding windows during WAL replay in stream computing
27. Made the output of USE DATABASE on macOS consistent with Linux; it now uniformly displays Database changed.
28. Fixed an internal logic error that caused OPC access failures on Windows when taosX bound to port 6051
29. Fixed a potential issue where the mnode message queue could become saturated during data compact operations if some nodes stayed offline for an extended period, causing connection exceptions
30. Fixed crashes in outer stream queries when diff() and notify were used together
31. Fixed a memory leak on taos_cleanup exit
